M365 Security Checklist 2026: 12 Steps to POPIA Compliance

Published April 2026 | 4 min read

Microsoft 365 is the #1 target for SA cyber attacks. Business Email Compromise cost South African companies R2.2 billion last year, and POPIA fines now reach R10 million. Most breaches we see could’ve been stopped with 30 minutes of settings.

We secured a 120-user manufacturer’s M365 tenant in 3 days using this exact checklist. Use it to audit your own setup, or send it to your IT guy.

The 12-Point M365 Lockdown Checklist

  1. Enforce MFA for all users, no exceptions – Use Microsoft Authenticator app. SMS is not secure. POPIA: You must protect account access.
  2. Block legacy authentication – Old mail apps bypass MFA. Attackers use this daily. Disable in Azure AD.
  3. Create separate admin accounts – Never use your daily email as Global Admin. One breach = total takeover.
  4. Set up Conditional Access – Block logins from Nigeria, Russia, China if you don’t operate there. Require compliant devices.
  5. Enable mailbox auditing – Turned off by default. You need this for POPIA breach investigations.
  6. Configure Safe Links & Safe Attachments – Stops 99% of phishing emails. Part of Defender for Office 365.
  7. Disable auto-forwarding to external emails – Common ex-employee/attacker tactic to steal data silently.
  8. Set up data loss prevention (DLP) – Auto-block ID numbers, bank details, CVs from being emailed externally. POPIA requirement.
  9. Review OAuth app permissions – Revoke “Mail.ReadWrite” access from dodgy apps. Attackers buy access tokens.
  10. Enable Unified Audit Log for 1 year – Default is 90 days. You need 1 year for POPIA compliance.
  11. Turn on Alerts: “Elevation of privilege” + “New global admin” – Get SMS/email if someone makes themselves admin.
  12. Immutable cloud backups for Email/OneDrive/SharePoint – Microsoft 365 recycle bin ≠ backup. Ransomware can delete it. Use third-party backup.
