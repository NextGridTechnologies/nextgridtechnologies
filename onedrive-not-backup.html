Stop Calling OneDrive Your "Backup". It's File Sync.

February 2026 · 5 min read · Backup

"But it's in the cloud!" — 3 reasons OneDrive will fail you when you need it most. Real cases from SA businesses.

Failure #1: Deleted Files Sync Deletion

Staff deletes folder locally → OneDrive syncs deletion to cloud. Recycle bin: 93 days max, then gone forever.

Real case: Architect deleted 2 years of client drawings. Recycle bin already emptied after 93 days. R200k project files lost.

Fix: True backup keeps versions even after deletion. We can restore files deleted 2 years ago.

Failure #2: Ransomware Syncs Encryption

Crypto-locker encrypts local files → OneDrive uploads encrypted versions. Version history overwritten if ransomware runs 30+ days undetected.

Real case: Retailer paid R80k ransom because OneDrive backups were encrypted too. No clean version to restore.

Fix: Immutable backup — can't be modified or deleted by ransomware for 30+ days.

Failure #3: Departing Staff = Data Loss

Employee leaves → Admin deletes M365 account → OneDrive deleted after 30 days. Manager asks for files 6 months later → Gone.

Real case: Sales director took client list, deleted OneDrive on exit. Company lost 5 years of CRM data.

Fix: Backup retains data even after account deletion. Legal hold for leavers.

What Real Backup Looks Like [3 Requirements]

  • Separate from production: Not in same M365 tenant. If Microsoft is down, backup still works.
  • Immutable: Can't be deleted or modified by admin or ransomware for X days.
  • Point-in-time restore: Pick any date in last 365 days, restore entire folder or single file.
